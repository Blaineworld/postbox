Escape Sequences

In programming, an escape sequence is a sequence of characters that changes the meaning of a character to something else. A typical example of this is a string in "quotation marks" where you want to have a quotation mark in the string, so you use \" to tell the computer that the quotation mark is just a quotation mark and isn't the end of the string. The same thing applies in Markdown, too. You can use a \ to say that a character is just a character and doesn't mean anything else. For example, you might need a # in a heading, so instead of # # My Heading, which results in a level two heading saying My Heading, you use # \# My Heading so that the # is treated as just part of the text. These can be used on pretty much anything that uses a character to represent a style.

Typographical Structures

Paragraphs

A paragraph, in case you somehow don't know, is a block of text that separates a piece of writing into smaller parts. In Markdown, you can create paragraphs by adding one or more lines of whitespace below a piece of text.

Headings

There are two ways to make headings in Markdown. The first one is with 1-6 #s depending on the level of your heading like # This is a top heading.. The "level" of a heading is basically just how small it is. The other is by underlining your heading with = or -, where = is the larger one.

Markdown Result

Test

Test

Test

 
Test

Test

Test

Test

Test

Quick Tip Headings are also given IDs based on their content so that you can refer to them using links. For example, the heading ### This is a heading. could be linked to using the link [Link to the heading.](#thisisaheading).

Block Quotes

A block quote is a quote set aside from the main article in its own rectangle, and you can create them by adding a > before each line of the quote. Block quotes can also have other block quotes inside of them, as well as lists and other elements.

Visual Example
Markdown Result 
> Reddit good
> Instagram bad
 
Reddit good
Instagram bad

Code Blocks

A code block is kind of like a block quote except that instead of a quote it's computer code and it cannot contain other elements or styles. You can write a piece of code inside of a line by `doing this`, or you can do what is shown in the visual example.

Lists

A list is a list of things. There are multiple types of lists you can use in Markdown, which you can see below.

Unordered Lists

An unordered list is a list that isn't numbered, which is why it's unordered. Each item on an unordered list is preceded by either * , + , or - .

Visual Example
Markdown Result 
- This is a list.
- Lists are nice.
  • This is a list.
  • Lists are nice.
 
- This is one marker.
+ This is another marker.
* ...and another!
  • This is one marker.
  • This is another marker.
  • ...and another!
Ordered Lists

An ordered list is a list that is numbered, which is why it's ordered. Each item on an ordered list is preceded by n. , where n is any number. You don't necessarily need to number the items correctly, as it will have no effect on the output.

Markdown Result 
**Things I Hate**
1. Markdown
2. lists
3. repetition
4. lists
5. irony
6. normies

  6. normies

  2. item two

Styles

Bold and Italics

Did you know italics are called that because they were invented in Italy? Sorry. Anyway, bold and italics are pretty simple to do in Markdown. To create italic text, you may surround your text with either *asterisks* or _underscores_. For bold text, simply do the same thing, but double the **asterisks** or __underscores__.

Visual Example
Markdown Result
_Hello world!_ Hello world!
*Hello world!* Hello world!
__Hello world!__ Hello world!
**Hello world!** Hello world!

Strikethrough

A strikethrough is when you don't like a piece of text so then you put a line through it and write something else in its place, but sometimes you want to add one intentionally. That's why Markdown has strikethrough, which you can apply using ~~double tildes~~ like so: ~~This text is scratched out.~~

Visual Example
Markdown Result
~~Fortnite good~~ Oops. Fortnite good Oops.

Emoji

Postbox allows you to use emoji, but how could that work if Postbox uses ASCII? The emoji on Postbox are PNGs loaded from a folder called emoji, so you can look in there to see all of the available emoji. The syntax for emoji is :e:, where e is the name of the emoji. Emoji should usually use dashes for spacing.
